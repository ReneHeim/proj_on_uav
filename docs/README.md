# Project Documentation

`docs/` is the canonical documentation root.

- `book/` retains the historical Sphinx/JupyterBook material, notebooks, and
  configuration files previously stored in `Documentation/`.
- `diagnostics/` contains focused operational notes such as GPU SIFT validation.
- Research run reports belong with their outputs under `outputs/runs/.../reports/`.
- Paper and slide deliverables belong under `outputs/deliverables/`.

Build the historical book with the command in the root `Makefile`.
